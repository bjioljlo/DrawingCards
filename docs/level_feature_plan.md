# Level（牌局階段）功能設計規劃

## 目標
設計 Level 類別，作為遊戲牌局的狀態管理核心，負責控制遊戲的各個階段，包括牌局開始、抽牌階段、行動階段、計算階段等，並提供階段切換和狀態查詢功能。

## Level 類別設計

### 屬性
- `CurrentPhase`：當前遊戲階段（enum GamePhase）
- `TurnCount`：當前回合數（int）
- `PhaseStartedAt`：當前階段開始時間（DateTime）
- `IsGameStarted`：遊戲是否已開始（bool）
- `IsGameOver`：遊戲是否結束（bool）

### 方法
- `StartGame()`：開始新牌局
- `NextPhase()`：進入下一個階段
- `EndGame()`：結束牌局
- `GetCurrentPhase()`：取得當前階段
- `CanProceedToNextPhase()`：檢查是否可以進入下一個階段
- `GetPhaseDuration()`：取得當前階段持續時間
- `Reset()`：重置牌局狀態

## 遊戲階段（GamePhase）
```csharp
public enum GamePhase
{
    NotStarted,      // 遊戲未開始
    Drawing,         // 抽牌階段
    Action,          // 行動階段
    Scoring,         // 計分階段
    GameOver         // 遊戲結束
}
```

## 勝利條件（VictoryCondition）
```csharp
public enum VictoryCondition
{
    None,           // 無條件（遊戲進行中）
    TargetScore,    // 達到目標分數
    NoCardsLeft,    // 無牌可抽
    SpecificCard,   // 抽到特定牌
    Custom          // 自訂條件
}
```

### 勝利條件檢查邏輯
1. **TargetScore（目標分數）**
   - 檢查所有玩家的分數
   - 如果任一玩家分數達到或超過目標分數，遊戲結束
   - 適用於：競速類遊戲、積分類遊戲

2. **NoCardsLeft（無牌可抽）**
   - 檢查牌組是否已空
   - 如果牌組和棄牌區都沒有牌，遊戲結束
   - 適用於：消耗類遊戲、記憶類遊戲

3. **SpecificCard（特定牌）**
   - 檢查是否有人抽到特定牌
   - 如果任一玩家手牌或棄牌堆中有特定牌，遊戲結束
   - 適用於：特殊任務類遊戲、關鍵牌遊戲

4. **Custom（自訂條件）**
   - 支援自訂勝利條件
   - 可根據遊戲規則靈活調整
   - 適用於：特殊規則遊戲、自訂模式
