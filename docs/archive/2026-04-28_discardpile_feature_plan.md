# DiscardPile（棄牌區）功能設計規劃

## 目標
設計 DiscardPile 功能，負責管理遊戲過程中被棄掉的牌，支援棄牌、查詢與補牌等操作。

## DiscardPile 類別設計（可用 List<Card> 或自訂類別）

### 屬性
- `Cards`：目前棄牌區的牌（IReadOnlyList<Card> 或 List<Card>）
- `Count`：棄牌區張數（int）

### 方法
- `Add(Card card)`：將牌加入棄牌區
- `AddRange(IEnumerable<Card> cards)`：將多張牌加入棄牌區
- `Clear()`：清空棄牌區（如補牌時）
- `GetLastCard()`：取得最上面一張棄牌（for UI 顯示）
- `GetAll()`：取得所有棄牌內容（for UI 彈窗顯示）

## 整合建議
- 棄牌區應與 Deck、Player、Hand 等類別互動，支援單張或多張棄牌與補牌。
- 遊戲流程中，Deck 為空時可將棄牌區洗牌補充回牌組。
- UI 可顯示棄牌區最上方的牌，並可點擊檢視全部棄牌容（彈窗）。

## 測試建議
- 驗證棄牌加入（單張、多張）與清空功能。
- 驗證補牌時棄牌區內容正確移動。
- 驗證查詢最上方棄牌與全部棄牌內容功能。

## 擴充性建議
- 支援多個棄牌區或特殊棄牌規則。
- 支援棄牌區內容查詢、統計。
