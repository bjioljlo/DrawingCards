# Table（桌面）功能設計規劃

## 目標
設計 Table 類別，作為遊戲中桌面區域的管理核心，負責管理桌面上的牌，支援牌的放置、移動和狀態同步等功能。

## Table 類別設計

### 屬性
- `MaxCards`：桌面最大可放牌數（int，可依遊戲規則調整）
- `CurrentCards`：目前桌面上的牌數（int）
- `Cards`：桌面上的所有牌集合（List<Card>）
- `IsFull`：桌面是否已滿（bool）

### 方法
- `AddCard(Card card)`：加入牌到桌面（若未達最大數量）
- `RemoveCard(Card card)`：將牌移出桌面（丟棄或其他用途）
- `SwapCards(int indexA, int indexB)`：交換桌面上兩張牌的位置
- `GetCards()`：取得目前桌面所有牌的清單
- `Clear()`：清空桌面所有牌
- `IsValidMove(List<Card> cards)`：驗證牌組是否可放置到桌面

## 整合建議
- Table 為玩家手牌、棄牌堆的中間過渡區，支援牌的移動操作。
- 需與 UI 同步，即時反映桌面狀態變化。
- 可根據遊戲規則調整最大牌數和放置規則。
- 支援多種牌的放置策略（如順序、組合等）。

## 測試建議
- 驗證牌的加入、移除、交換操作的正確性。
- 驗證桌面最大牌數限制。
- 驗證牌的狀態同步。
- 驗證特殊情況處理（如桌面已滿、無牌可移等）。

## 擴充性建議
- 支援多種桌面布局（如線性、網格等）。
- 支援自訂放置規則（如特定組合、順序等）。
- 支援桌面區域的擴展（如增加特殊區域）。
- 支援多種牌的放置策略（如自動排列、手動調整等）。
